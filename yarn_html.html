Premier point

Qu'est-ce que Yarn

Yarn est un webpack manager. C'est-à-dire qu'il installe et gère
les webpack à la place du client



Second point

Pourquoi Yarn plutôt que NPM

Yarn a été développé par Facebook et Google. Il est plus rapide
et stable que NPM mais surtout, une fois un package installer,
il peut être utilisé en Offline. NPM installe les packages
les uns après les autres (sequetially) la où Yarn les installe
en parallèle (in aprallel)



commandes Yarn

Pour init un projet

yarn (ou yarn install) --> installer les dépendances depuis le package.json


on reviendra sur le package.json plus tard

yarn add --> ajoute un package au projet dans le dossier et crée les dépendances

yarn upgrade --> Met à jour les listes de dépendances si on change de version
dans le package.json

yarn global add --> Installe un package sur le système d'exploitation de manière général

yarn remove --> Supprime une/des dépédance/s et leur/s références dans le package.json



Vous allez maintenant installer yarn depuis NPM : "npm install yarn --global" dans la console

Verifiez aussi la version de yarn : "yarn --version"



Qu'est-ce que le package.json ?

Le package.json est en quelques sortes le tableau éléctrique du site,
soit l'endroit avec tout le cablage et les fusibles (dans ce cas, les différentes
versions des packages)
